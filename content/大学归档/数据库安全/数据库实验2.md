# 数据库安全实验2

熟悉单向哈希函数以及消息认证码（MAC），使用工具或者编程为一段指定的信息生成哈希值以及消息认证码。（使用Linux系统的openssl命令行工具及开发库）

## 前言

其实现使用了实验1的Win64OpenSSL。注意，您可以在Linux里随意复现这些操作。

## 进行测试

使用-md5,-sha1,-sha256算法，分别进行计算，得到消息摘要：

```bash
openssl dgst -md5 nonamefile.txt
openssl dgst -sha1 nonamefile.txt
openssl dgst -sha256 nonamefile.txt
```

结果如图所示：

![image-20220509005631352](https://pic.firehomework.top/2025/03/3e64fe0d8c4749902b0748efc2dbeb9c.webp)

接下来进行修改：在文件最后添加一个字符：

![image-20220509005652335](https://pic.firehomework.top/2025/03/8a76260bace6d48abb7781ac84a5f506.webp)

重新运行上面的命令：

![image-20220509005720029](https://pic.firehomework.top/2025/03/c4c4b57aaeb1477652a7f7d9a31fdb16.webp)

可以看到，之前和之后的文件，消息摘要发生了巨大的变化。

## 生成消息认证码

```bash
openssl dgst -md5 -hmac "随便输入" nonamefile.txt
```

![image-20220509005824426](https://pic.firehomework.top/2025/03/66b62ef2991ad51823f1ef1ad0f40ae3.webp)

我们对参数进行一些修改：

```bash
openssl dgst -md5 -hmac "随便输的" nonamefile.txt
```

![image-20220509005853303](https://pic.firehomework.top/2025/03/5d94eeeeface3c60e4da459d3dc3bb0c.webp)

可以发现和之前不同并发生较大变化。

将对应的参数进行删减：

```bash
openssl dgst -md5 -hmac "的" nonamefile.txt
```

![image-20220509005924795](https://pic.firehomework.top/2025/03/cf53d28f5dd3a0c01f4fb34afc03b477.webp)

生成的串长度不变。
